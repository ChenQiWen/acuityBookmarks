# Frontend 环境变量说明

## ⚠️ 重要提示

**此目录不再维护独立的 `.env` 文件！**

所有环境变量已统一管理在项目根目录：

- **根目录 `.env`**：公共配置（可提交）
- **根目录 `.env.local`**：本地开发配置（不提交）

## 为什么这样做？

1. **避免重复定义**：多个 `.env` 文件容易造成配置不一致
2. **统一管理**：所有配置集中在根目录，便于维护
3. **Vite 配置**：`vite.config.ts` 已配置从根目录加载环境变量

## 如何配置环境变量？

### 生产环境配置

编辑根目录的 `.env` 文件：

```bash
# 根目录 .env
VITE_API_BASE_URL=https://api.acuitybookmarks.com
VITE_CLOUDFLARE_WORKER_URL=https://acuitybookmarks.cqw547847.workers.dev
```

### 本地开发配置

编辑根目录的 `.env.local` 文件（不提交到 Git）：

```bash
# 根目录 .env.local
VITE_API_BASE_URL=https://localhost:8787
VITE_CLOUDFLARE_WORKER_URL=https://localhost:8787
```

## 环境变量加载优先级

```
.env.local（本地开发）
    ↓ 覆盖
.env（公共配置）
    ↓ 默认值
代码中的降级逻辑
```

## 相关文件

- `vite.config.ts`：Vite 配置，指定从根目录加载环境变量
- `src/config/constants.ts`：环境变量使用和降级逻辑
